# 修复KB_scan和KB_scan_report执行问题并添加INFO日志

## 问题分析

1. **输出未捕获**：KB_scan.py使用`print`语句而不是`logger`，导致输出没有被捕获到主日志中
2. **错误处理不足**：vm_management_workflow.py中启动KB_scan和KB_scan_report时没有充分的错误处理
3. **执行流程不可见**：KB_scan和KB_scan_report缺少详细的INFO日志，无法看到执行进度
4. **命令行参数传递**：可能存在命令行参数传递或工作目录的问题

## 修复计划

### 1. 修改KB_scan.py
- 将所有`print`语句替换为`logger.info`语句
- 添加详细的INFO日志，标记每个执行步骤
- 确保正确处理命令行参数
- 添加异常处理，确保错误能够被捕获并记录

### 2. 修改kb_scan_report.py
- 添加更多的INFO日志，标记每个执行步骤
- 确保正确处理命令行参数
- 增强错误处理，确保错误能够被捕获并记录

### 3. 修改vm_management_workflow.py中的run_kb_scan_and_report函数
- 添加更多的错误处理和日志输出
- 确保命令行参数传递正确
- 捕获并记录KB_scan和KB_scan_report的执行结果
- 添加工作目录检查，确保文件路径正确

### 4. 增强日志记录
- 在关键执行点添加INFO日志
- 确保日志格式一致，便于分析
- 添加执行时间戳，便于追踪执行顺序

## 预期效果

修复后，当运行vm_management程序时，应该能够：
1. 看到KB_scan和KB_scan_report的详细执行流程
2. 捕获并记录所有错误信息
3. 确保命令行参数传递正确
4. 提供清晰的日志输出，便于分析执行结果

这样可以大大提高程序的可维护性和可调试性，确保KB扫描和报告生成过程能够正常执行并提供详细的执行状态。
